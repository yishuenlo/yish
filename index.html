Main takeaways:

  • JSX is not Javascript nor HTML. Its syntax is like HTML, but it's an extension of Javascript. And you write JSX in JavaScript file.
  • You need "translator" to compile and translate JSX into Javascript. (Something like Babel? I needed to install Node for React to work... need to learn this later.)
  • React and ReactDOM are different. React is a JS library for buildig UI, ReactDOM is the JS library for React to interacte with DOM. You need to import both into JavaScript file for React to work properly.
  • React uses Virtual DOM to reduce re-rendering DOM from scratch everytime something is updated. Virtual DOM is lightweight, so React compares current version of Virtual DOM from previous version to figure what changed, and only update the content that changed from previous version.
  • Analogy for Virtual DOM = Home Improvements: update and compare CAD/Sketchup plans, and remodel specific rooms that changed instead of rebuilding the whole house from scratch every time you want to do home improvements.
  • Call Javascript within JSX with { }.

$41 // November 24, 2019

What is command line and why should I use it?

What is the Command Line? Command Line a simple, basic way to communicate with our computers. Our computers have evolved to use Graphic User Interface for users to communicate with computers - such as Window Explorers. GUI is intuitive and visual, and requires mostly mouse clicks. On the other hand, Command Line only uses keyboard and text.

Why should I use the Command Line? Command Line offers more ways to communicate with my computer through one channel, versus jumping through different locations and window explorers. For example, Git uses command line. What it does is that the Command Line has ALL control of Git, whereas if I use a GUI version, I'd have to install Github and uses Github's app to access different versions of Git. While Github is more intuitive, it doens't offer more power and control.

One example of how Github doesn't have more control is when I had to use different Github users. Switching through Github is hard and confusing... and almost seems impossible. Just becasue I logged out of Github doesn't mean my computer is unlinked from my previous Git source. So while my Github is saying I've logged out of loys73 and is now using yishuenlo, my computer's Git is still linked to loys73.

Apparently using command line is also easier to many batches of files, such as changing file names just for .txt file. In summary, Command Line is the most rudimentary way to communicate with computers, but also the easier way to control and let communters understand what we need from them. Almost like comparing 文言文 and 白話文.

Study the basics of creating parallax using CSS.

Main takeaway: use background properties to create parallax effects. background-attachment is the backbone of parallax effect. Use fixed to make background fixed relative to hte viewpoert, so it doesn't move with other elements.

background-attachment: fixed;
background-position: center;
background-repeat: no-repeat;
background-size: cover;
